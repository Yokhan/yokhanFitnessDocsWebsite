# Domain: Science & Reasoning — Evidence-Based Guard

## NEVER Recommend (Methodological Errors)
1. **P-Hacking** — Running analyses until p<0.05 inflates false positives to ~64% with 20 metrics. Instead: pre-register, correct for multiple comparisons.
2. **HARKing** — Hypothesizing after results are known converts exploratory to false "confirmatory." Instead: separate exploratory vs confirmatory; pre-register.
3. **Publication Bias** — Positive results published preferentially; effect sizes inflated systematically. Instead: registered reports, publish null results.
4. **Confusing Correlation With Causation** — Observational correlations are not causal evidence. Instead: RCTs, natural experiments, causal inference methods.
5. **Cherry-Picking Evidence** — Selecting studies that confirm beliefs, ignoring contradictory ones. Instead: systematic reviews, pre-registered search protocols.
6. **Small Sample Overgeneralization** — n=12 studies treated as definitive. Instead: require replication, report effect sizes with confidence intervals.
7. **Survivorship Bias** — Studying only successes (surviving companies, published studies). Instead: actively seek failure data, non-published results.
8. **Appeal to Nature Fallacy** — "Natural = safe/good" is false (arsenic is natural; vaccines are artificial). Instead: evaluate evidence regardless of source.
9. **Single Study Syndrome** — One study, even well-done, is hypothesis-generating, not proof. Instead: require replication and meta-analytic evidence.
10. **Base Rate Neglect** — Ignoring prevalence when interpreting test results (99% accurate test with 1% prevalence = 50% false positives). Instead: always apply Bayes' theorem.
11. **Anchoring Bias** — First number encountered dominates estimates even when irrelevant. DETECT: initial value disproportionately influences final judgment. Instead: generate estimate BEFORE seeing others; use multiple independent estimates.
12. **Framing Effect** — Same data presented differently leads to different decisions (90% survival vs 10% mortality). Instead: present data in multiple frames; use absolute numbers alongside percentages.
13. **Availability Heuristic** — Judging probability by ease of recall, not actual frequency (plane crashes vs car crashes). Instead: look up base rates; ask "is this vivid or actually common?"
14. **Conjunction Fallacy** — Judging "A and B" as more likely than "A" alone when B adds narrative coherence (Linda problem). Instead: decompose compound claims; check if adding detail reduces probability.
15. **Ecological Fallacy** — Inferring individual traits from group-level data (rich countries are happy, therefore rich people are happy). Instead: match analysis level to claim level; use individual-level data for individual claims.

## ALWAYS Apply (Reasoning Principles)
1. **Pre-Registration** — Declare hypotheses, methods, and analysis plan before collecting data. Prevents p-hacking and HARKing.
2. **Effect Sizes Over P-Values** — P-values say "is there an effect?" Effect sizes say "how big?" Always report both with confidence intervals.
3. **Replication as Standard** — No finding is established until independently replicated. Budget for replication in study design.
4. **Bayesian Reasoning** — Update beliefs proportional to evidence strength. Prior probability matters. Use Bayes' theorem for diagnostic decisions.
5. **Evidence Hierarchy** — Systematic reviews > RCTs > cohort > case-control > case series > expert opinion. Weight advice accordingly.
6. **Steelmanning** — Engage with the strongest version of opposing arguments, not strawmen. Reduces confirmation bias.
7. **Falsifiability** — Claims that cannot be proven wrong are not scientific. Demand testable predictions.
8. **Multiple Competing Hypotheses** — Generate 3+ explanations for any observation before testing. Prevents anchoring on first theory.
9. **Pre-Mortem Analysis** — Before deciding, imagine the decision failed. What went wrong? Surfaces risks that optimism bias hides.
10. **Distinguish Absence of Evidence from Evidence of Absence** — "No studies found X" differs from "studies found no X." The distinction is critical.
11. **Bayesian Update Formula** — Posterior = Prior x Likelihood / Evidence. For practical use: extraordinary claims require extraordinary evidence; mundane claims require less.
12. **Sensitivity Analysis** — Test how conclusions change when assumptions vary. If result flips with small assumption changes, confidence should be low.
13. **Debiasing via Reference Class Forecasting** — Instead of inside view (this project's specifics), use outside view (how similar projects actually performed). Kahneman: reduces planning fallacy by 30-50%.
14. **Consider the Opposite** — Before finalizing a judgment, actively argue the opposing case. Lord, Lepper & Preston (1984): reduces confirmation bias measurably.
15. **Calibration Training** — Practice making probabilistic predictions and tracking accuracy. Well-calibrated forecasters (Tetlock's superforecasters) update frequently in small increments, not big swings.

## Evidence Enforcement (MANDATORY for scientific claims)

Every scientific claim or recommendation MUST include:
- **Evidence level**: A (systematic review/meta-analysis) / B (RCT/large cohort) / C (expert consensus) / D (blog/opinion)
- **Source**: author/org, year, publication
- If level D or no source: state "based on general patterns, not established evidence" — never assert as fact.
- When presenting conflicting evidence: state both sides, note evidence levels, let user decide.
