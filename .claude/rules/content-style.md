# Стайл-гайд контента

## Голос автора

Игорь — фитнес-тренер с бэкграундом в gamedev и IT. Пишет для разработчиков, дизайнеров, инженеров — людей с сидячим образом жизни. Тон — как разговор с другом: дружеский, без менторства, с юмором.

## Правила текста

- Обращение на «вы» (основное), иногда «ты» в неформальных вставках
- Отсылки к играм (DOTA, ранкед, видеоигры), IT-работе и сидячему образу жизни — это фирменный стиль
- Геймерские метафоры приветствуются: «+10 к настроению», «не баг, а фича»
- Эмодзи — в меру, обычно в конце раздела или заголовка (😁, 😉)
- Без формального/академического тона. Без канцелярита
- Без фармы, «химии», бега и прыжков — автор это подчёркивает

## Структура документа

- H1 — из `title` в frontmatter (не писать вручную)
- H3 (`###`) — основной рабочий уровень заголовков
- H2 (`##`) — редко, для крупных разделов
- Списки — буллеты предпочтительнее нумерованных
- `**жирный**` — для ключевых терминов и понятий
- Цитаты (`>`) — для жизненных примеров и сценариев

## Адмониции Docusaurus

Использовать активно. Формат с русским заголовком в скобках:

```
:::note[Определение]
Текст заметки
:::

:::tip[Не баг, а фича!]
Текст совета
:::

:::info[Заголовок]
Информационный блок
:::

:::warning[Внимание]
Предупреждение
:::

:::danger[Чего не будет]
Запрет или предостережение
:::
```

## Frontmatter

Все поля обязательны. Копируй шаблон из CLAUDE.md. Не удаляй `obsIMG` и `free` — legacy-поля, автор их использует.

## Чего НЕ делать в контенте

- Не использовать `[[wiki-ссылки]]` — Obsidian-синтаксис, не работает в Docusaurus
- Не писать H1 вручную (берётся из title)
- Не менять слаги существующих страниц
- Не добавлять английский текст в docs/
- Не использовать формальный/научный стиль
